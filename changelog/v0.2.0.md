## @globio/sdk v0.2.0

### What's new

**GlobioPulse** — Live configuration and feature flags. Fetch all configs and 
flags for an environment, get individual config values with full TypeScript 
generics, and check feature flag state. Results are cached client-side for 
5 minutes to avoid unnecessary API calls.

**GlobioScope** — Game analytics. Track custom events with properties, batch 
tracking with automatic flush every 2 seconds or 20 events, and force-flush 
for critical events like purchases. Events are fire-and-forget — tracking 
never blocks your game loop.

### Usage
```typescript
// Feature flags
const isNewMapEnabled = await globio.pulse.isEnabled('new_map');
const maxPlayers = await globio.pulse.get<number>('max_players');

// Analytics
globio.scope.track('level_completed', { level: 5, time_seconds: 120 });
globio.scope.track('item_purchased', { sku: 'sword_of_fire', currency: 'GEMS' });

// Force flush for critical events
await globio.scope.flush();
```

### Notes
- GlobioPulse results are cached for 5 minutes — call `globio.pulse.clearCache()` 
  to force a fresh fetch
- GlobioScope batches automatically — do not await `track()`, it is intentionally 
  synchronous at the call site
- For critical economy events, always call `await globio.scope.flush()` immediately 
  after tracking to ensure the event is not lost if the browser closes
